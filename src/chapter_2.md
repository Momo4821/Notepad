# Chapter 2 What I've Learned



## Thoughts

- This was my first time building a full WPF application, combining UI design with backend C# logic. I learned how WPF controls behave, how dialogs work, and how application state flows through the UI.
- I gained a better understanding of clean code principles: keeping responsibilities separated, avoiding redundant data, and centralizing logic where it makes sense.
- Debugging taught me how important timing and state management are in UI applications. Many issues weren’t “bugs” — they were misunderstandings of when certain values actually exist.
- I didn’t use AI during development, but afterward I used it as a technical sounding board to validate architectural decisions and clarify areas that still felt confusing.
- Building this Notepad clone was a valuable first project. It gave me hands‑on experience with real UI patterns and helped me think like a developer. I hope this documentation helps other beginners see both the technical side and the learning process behind the app.
